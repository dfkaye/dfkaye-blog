This paragraph is a <p> tag in a content file with the ".html" extension.

Early on I found that creating demos with application code requires inserting raw HTML into the markdown content, using this shortcode by Ana Ulin.

That approach allows you to include localization attributes in HTML tags, en français, for example. 


<i lang="fr-FR">en français</i>

Alternatively, you can name the content file with the ".html" extension instead, and include the usual front matter at the top of the content file, like so: 


---
title: "Hugo content as HTML"
date: 2020-07-08T13:40:43-07:00
draft: false
---

<p>
  This paragraph is a <code><p></code> tag in a content.html file.
</p>

...

However, by using the ".html" extension in Hugo, you lose the processing power of partials. That's why the code blocks above are not styled with Hugo's code-fences as they are on msot other pages here.

Anyway, food for thought…
